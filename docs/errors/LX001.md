# Error LX001: Unterminated String Literal

This error occurs when a string literal is opened with a quote but the file ends or a line breaks before it is closed.

## Incorrect Code Example

```arandu
x = "hello
```

## Semantic Explanation

The lexical analyzer scans the source code byte by byte. When it encounters a quotation mark, it starts capturing characters for a string literal until it finds the matching closing quotation mark. If it reaches the end of the line or the file (EOF) without finding it, it raises a lexical error.

## How to Fix

Ensure every string literal is properly terminated with matching quotation marks:

```arandu
x = "hello"
```
