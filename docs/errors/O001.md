# Error O001: Use of Moved Value

This error occurs when you try to read or use a variable after its value has been moved (ownership transferred) to another variable or passed as a value argument to a function.

## Incorrect Code Example

```arandu
struct Data {
    value i32
}

func process(d own Data) {
    // d is consumed here
}

func main() {
    x = Data { value: 42 }
    process(x) // x is moved to the process function
    y = x.value // Error O001: use of moved value: 'x'
}
```

## Semantic Explanation

Arandu uses a single-ownership model for memory management. When a variable containing a non-copyable structured resource is passed to a function or assigned to another variable, ownership of the resource is transferred (moved). After this point, the original variable becomes invalid/deallocated and can no longer be read by the compiler to prevent issues such as use-after-free.

## How to Fix

### 1. Borrow by Reference

If the function does not need exclusive ownership of the resource, change its signature to accept a shared reference (`ref`) or a mutable reference (`ref mut`), allowing the original variable to remain usable after the call:

```arandu
struct Data {
    value i32
}

func process(d ref Data) {
    // d is only borrowed
}

func main() {
    x = Data { value: 42 }
    process(x) // x is borrowed by reference
    y = x.value // Correct
}
```

### 2. Clone the Value

If the function genuinely requires exclusive ownership but you also need to use the data afterwards, make an explicit copy or clone of the resource:

```arandu
struct Data {
    value i32
}

func process(d own Data) {
    // d is consumed here
}

func main() {
    x = Data { value: 42 }
    xClone = x.clone() // Duplicate the object in memory
    process(xClone)
    y = x.value // Correct: x was not moved, its clone was
}
```
