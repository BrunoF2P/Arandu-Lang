# Note O004: Generational Fallback

**Severity: informational (not a hard error).**  
The compiler detected that a stack local **escapes** a statically limited borrow
window (return, heap store, or similar) and either refused the pattern or
inserted a **controlled generational/escape fallback** so the program remains
memory-safe.

## Example Situation

```arandu
func escape(): &int {
    let x = 1
    return &x // escapes stack frame → O004 / O010
}
```

## Semantic Explanation

Arandu prefers stack-first ownership without surface lifetimes. When the live
range of a reference cannot be closed inside the current CFG, the compiler must
not invent silent heap magic: it reports **O004** (and may pair **O010** when
the program is rejected) so the decision is inspectable (“Magia Inspecionável”).

## How to Fix (stack-first)

- Keep the owner alive in the caller and pass `&` only within that scope
- Return owned data instead of a reference to a local
- Use an explicit allocator / heap type when escape is intentional

This code is reserved for **G2 / F2.3** diagnostics; it is **not** used for
shared-borrow conflicts (those are **O003**).
