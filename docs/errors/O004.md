# Error O004: Shared Borrow Conflict

This error occurs when attempting to mutate a variable while it is borrowed shared.

## Incorrect Code Example

```arandu
let x = Data {}
let r = &x
set x.value = 5 // Conflict
```

## Semantic Explanation

Shared references guarantee that the data remains read-only for their duration. Mutating the underlying variable violates this guarantee.

## How to Fix

Do not mutate the variable while references are alive:

```arandu
let x = Data {}
set x.value = 5
let r = &x // Borrow after mutation
```
