# Note O004: Generational Fallback

**Severity: informational note by default** (`Severity::Note`).  
Not a silent compiler mode change. The note is always visible when the static
borrow window is insufficient (“Magia Inspecionável”).

## When it appears

The compiler detected that a stack local **escapes** a CFG-limited borrow window
(return, heap store, or similar). Static F2.2 liveness cannot close the loan.

- **Return of `&local`**: always also emits **O010** (hard error — a stack frame
  cannot be gen-fixed after return).
- **Other escapes** (e.g. store into memory that may outlive the frame): O004
  records the decision; future runtimes may insert Vale-style generation checks.

## G2 — Promote O004 to a hard error (opt-in)

G2 is fused into F2.3 as **explicit opt-in**, not a silent “strict mode”:

```arandu
@no_fallback
func critical(): int {
    // any O004 in this function becomes Severity::Error
    ...
}
```

CLI (same atomic):

```text
arandu_cli check path.aru --no-generational-fallback
# or: -Zno-generational-fallback
```

Default compilation still emits O004 as a **note** so developers see *why* a
value left the stack-first path, without changing success/failure unless they
opt in.

## Example

```arandu
func escape(): &int {
    let x = 1
    return &x // O010 + O004 note
}
```

## How to stay stack-first

- Keep the owner alive in the caller and pass `&` only within that scope
- Return owned data instead of a reference to a local
- Use an explicit allocator / heap type when escape is intentional

**Not** used for shared-borrow conflicts (those are **O003**).
