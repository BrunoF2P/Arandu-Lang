# Error T018: Field Does Not Exist on Type

This error occurs when you try to access a field on a struct (or union) that is not defined in its declaration.

## Incorrect Code Example

```arandu
struct User {
    age: int
}

func main() {
    u = User { age: 30 }
    x = u.name // Error T018: no field 'name' on type 'User'
}
```

## Semantic Explanation

In Arandu, all structs have static, rigid fields determined at compile time. Unlike dynamic languages, you cannot access or assign to fields that are not explicitly declared in the `struct` block. The type checker analyzes the access `u.name` and compares it against the fields defined in `User` (in this case, only `age`).

## How to Fix

### 1. Correct the Field Name (Typo)

If the field exists but you misspelled it, correct the spelling. The Arandu compiler uses Levenshtein distance to suggest the closest field:

```arandu
struct User {
    age: int
}

func main() {
    u = User { age: 30 }
    x = u.age // Correct
}
```

### 2. Add the Field to the Struct Definition

If the field is indeed supposed to exist on the data structure, add it to the struct definition:

```arandu
struct User {
    age: int
    name: str
}

func main() {
    u = User { age: 30, name: "Bruno" }
    x = u.name // Correct
}
```
