# Error T030: Duplicate field declaration

A struct was declared with multiple fields sharing the same name.

## Erroneous Code Example

```arandu
struct Point {
    x: i32,
    y: i32,
    x: f32, // error: field 'x' is already declared in struct 'Point'
}
```

## Semantic Explanation

Each field in a struct must have a unique name within the struct scope. Duplicate field names create ambiguity when accessing struct properties and prevent layout generation.

## How to Fix

Verify that all field names are spelled correctly and represent unique properties:

```arandu
struct Point {
    x: i32,
    y: i32,
    z: f32, // ok!
}
```
