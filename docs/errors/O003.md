# Error O003: Mutable Borrow Conflict

This error occurs when two borrows of the same place **overlap in time** and at
least one of them is exclusive (`&mut`), or when the place is mutated while a
shared borrow is still live.

Shared + shared overlap is **allowed** (read aliasing is safe).

## Incorrect Code Example

```arandu
mut x = 0
let a = &mut x
let b = &x // O003: exclusive conflict
```

```arandu
mut x = Data {}
let r = &x
set x.value = 5 // O003: mutation under shared borrow
```

## Semantic Explanation

Exclusive borrows require unique access. Another shared borrow, another exclusive
borrow, or a mutation of the place while the exclusive (or shared) borrow is live
violates that uniqueness.

## How to Fix

End the first borrow (let the reference go out of use) before creating a
conflicting one, or avoid mutating while shared references exist.
