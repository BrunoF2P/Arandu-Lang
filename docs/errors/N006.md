# Error N006: Import Name Conflict

An import introduces a name that is already bound in the same scope (another
import or declaration). The second binding is rejected so resolution stays
unambiguous.

## Incorrect Code Example

```arandu
import std.path as path
import my_app.path as path

func main(): int {
    return 0
}
```

## Correct Code Example

```arandu
import std.path as path
import my_app.path as local_path

func main(): int {
    return 0
}
```

## Notes

- Also applies to named imports that bind the same local identifier twice.
- Prefer distinct aliases when importing similar APIs from two modules.
