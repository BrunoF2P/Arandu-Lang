# Error LX003: Invalid Number Literal

This error occurs when a numeric literal has an invalid format, such as multiple decimal points or invalid digit characters for its radix.

## Incorrect Code Example

```arandu
x = 1.2.3
y = 0b12
```

## Semantic Explanation

Numeric literals must follow strict lexical formats (e.g., standard decimal, binary with 0b, octal with 0o, hex with 0x). The lexer validates digits against the selected base.

## How to Fix

Correct the format of the number literal:

```arandu
x = 1.23
y = 0b10
```
