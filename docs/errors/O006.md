# Error O006: Dangling Reference

This error occurs when a reference outlives the variables or data structure owning the underlying memory.

## Incorrect Code Example

```arandu
mut r ptr[int] = nil
{
    x = 5
    set r = &x
} // x goes out of scope
// r is now dangling
```

## Semantic Explanation

References must always point to valid memory. If the owner of the memory goes out of scope, references to it must terminate.

## How to Fix

Extend the scope of the owner or limit the lifetime of the reference:

```arandu
x = 5
r = &x
```
