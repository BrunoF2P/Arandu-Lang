# Error O006: Destroy While Borrowed

This error occurs when storage is **destroyed, freed, or otherwise invalidated**
while a borrow of that storage is still active. Catching this statically is what
makes user-facing double-free a compile error instead of relying only on runtime
poison checks.

## Incorrect Code Example

```arandu
mut x = alloc(0) // conceptual
let p = &x
free(x) // O006 if free/destroy runs while `p` is live
use(p)
```

## Semantic Explanation

Destroying an owner ends the validity of every outstanding reference. Emitting
`Destroy` / `free` / drop while a borrow window is open would create a dangling
reference or double-free risk.

## How to Fix

End all borrows (let references go dead) before destroy/free, or restructure
ownership so the destroy is in a branch where no borrow is live.

After M2 is complete, runtime poison (`0xDE`) remains only as a **compiler bug
net**, not the primary user-facing check.
