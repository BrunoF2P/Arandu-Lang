# Error T026: Cannot assign twice to immutable variable

An attempt was made to reassign a value to an immutable local variable or parameter. In Arandu, variables are immutable by default and must be explicitly declared as mutable using the `mut` modifier to allow reassignment.

## Erroneous Code Example

```arandu
fn main() {
    let x = 5;
    x = 10; // error: cannot assign twice to immutable variable 'x'
}
```

## Semantic Explanation

By default, bindings are immutable. This design choice guarantees safety by preventing accidental state modifications and enables simpler concurrency reasoning. To modify a variable's value after its initial binding, the variable must be marked with `mut`.

## How to Fix

Declare the variable using the `mut` modifier:

```arandu
fn main() {
    let mut x = 5;
    x = 10; // ok!
}
```
