# Error O002: Move While Borrowed

This error occurs when code tries to **move or consume** a value while it still
has an active borrow (`&` / `&mut` or an equivalent method receiver window).

## Incorrect Code Example

```arandu
let user = User { name: "Ada" }
let p = &user
// ...
consume(user) // O002: move while borrowed
use(p)
```

## Semantic Explanation

An active borrow guarantees that the owner’s storage stays valid for the
borrow’s lifetime (the live range of the reference). Moving the owner would
invalidate that storage, so the move is rejected at compile time.

## How to Fix

- Finish using the borrow before moving, or
- Clone/copy if the type allows, or
- Restructure so the move happens outside the borrow window.

```arandu
let user = User { name: "Ada" }
let p = &user
use(p)
// borrow of `p` ends
consume(user)
```
