# Error O002: Borrow After Move

This error occurs when trying to borrow or reference a variable after its value has already been moved.

## Incorrect Code Example

```arandu
let x = Data {}
process(x) // Move
let ref_x = &x // Borrow after move
```

## Semantic Explanation

Once a value's ownership is moved, its memory state is invalid. Borrowing a moved variable would create a dangling reference.

## How to Fix

Borrow the value before moving it, or avoid moving it if you still need it:

```arandu
let x = Data {}
let ref_x = &x
```
