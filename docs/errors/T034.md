# Error T034: Cannot Format Value as `str`

This error occurs when a value is used where a `str` is required (string
interpolation or a call argument typed as `str`) but the value's type cannot
be auto-formatted in ToStr v0.1.

## Incorrect Code Example

```arandu
struct Point {
    x: int
    y: int
}

func main() {
    let p = Point { x: 1, y: 2 }
    io.println(p)
    let s = "point=${p}"
}
```

## Semantic Explanation

ToStr v0.1 supports automatic formatting only for:

- `str` (identity)
- `bool`, `char`
- integer types (`int`, `uint`, fixed-width integers, `byte`)
- float types (`float`, `f32`, `f64`)

User-defined types, `Result`, `Option`, pointers, slices, and arrays are not
formatable yet. Full user `Display` / custom `to_str` is planned for a later
version.

## How to Fix

Format the value yourself into a `str` (e.g. print fields individually), or
pass a value of a supported primitive type:

```arandu
func main() {
    let n: int = 42
    io.println(n)
    let s = "n=${n}"
}
```
