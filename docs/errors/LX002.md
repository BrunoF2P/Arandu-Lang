# Error LX002: Invalid Unicode Character

This error occurs when the source file contains characters that are not allowed outside string literals or comments.

## Incorrect Code Example

```arandu
x @ = 10
```

## Semantic Explanation

Arandu accepts standard UTF-8 source files, but identifiers and operators are restricted to specific Unicode sets. Arbitrary control characters or special glyphs are rejected by the lexer.

## How to Fix

Remove the invalid character or place it inside a comment or string literal:

```arandu
x = 10 // @ symbol removed
```
